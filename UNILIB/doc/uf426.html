UF426

[ Name | Synopsis | Arguments | Description | Diagnostics | Common Blocks | Dependencies | See Also | Reported Bugs | Examples ]

Name

    Subroutine UF426 - interpolating an extremum of B

Synopsis 

      
      REAL*8    bextr, aextr
      INTEGER*4 ifail, ktype  
          
      CALL UF426 (bextr, aextr, ktype, ifail)

Arguments

    bextr
    Magnetic field intensity at the extremum [out]
    aextr
    Altitude at the extremum [out]
    ktype
    Type of extremum (-1; +1) [out]
    ifail
    error flag or extremum index (see diagnostics) [out]

Description

    The subroutine UF426 interpolates an extremum of the magnetic field intensity along a magnetic field line. To be used, the common block UC170 has to include at least three points and the last Runge-Kutta step had to overreach the extremum.

    The subroutine UF426 combines parabolic interpolations and Runge-Kutta steps to determine the correct location of the extremum. Note that the two last points stored in the common block UC170 can be modified and that a new point can be added by the subroutine.

    When the subroutine succeeds, the arguments bextr and aextr are set to the magnetic field intensity and altitude at the extremum location, respectively. The argument ktype is set to 1 when the extremum is a maximum and to -1 for a minimum. The index in the common block UC170 corresponding to the extremum is stored in the argument ifail.

    Note that the subroutine takes into account the step size limit stplst defined in the common block UC190.

Diagnostics

    -42601, two points are identical
    -42602, B does not vary
    -42603, too many iterations
    -42604, -42605, problem with the Runge Kutta step
    -42606, too many iterations

Common Blocks

    UC160, general constants
    UC170, temporary magnetic field line
    UC190, control parameters, set 1

Dependencies

Called by

    UF422, follow a field line until a boundary condition is reached

Calls

    UF423, Runge Kutta step

See also

    None

Reported Bugs

    18-Dec-97(v1.11), Malfunction of subroutine UF426

Examples

    None
