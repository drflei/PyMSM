Bug Report

[ Signature | Diagnostic | Solution | History ]

Signature

    R.02 - Erroneous behaviour of most of the external magnetic field models

Diagnostic

    An erroneous behaviour of the library occurs under some Unix operating systems with the external magnetic field models:
    • Tsyganenko short [1987],
    • Tsyganenko long [1987],
    • Tsyganenko [1989], and
    • Olson & Pfitzer quiet [1977].
    It has been detected by the comparison of results obtained under HPUX and VMS with the version 1.06 of the library.

    The listed external magnetic field models are implemented in the package EXT530 based on the source code provided by the respective authors. Unfortunately some subroutines make use of the static property of the FORTRAN variables to initialize variables only the first time that the subroutine is called. Under UNIX, the F77 compiler makes use sometimes of dynamic variables by default, and the FORTRAN statement SAVE has to be used explicitely.

    The problem does not seem to appear under SunOS and DEC/OSF.

Solution

    These external magnetic field models should not be used with version 1.06 or earlier of the library.

    In the next versions, FORTRAN SAVE statements have been added in the function TKSI and in the subroutines BXYZMU, DIPOLE, INTERCON, R2_BIRK, TSY87S, TSY87L and TSY89.

History

    September 10, 1997 - Bug reported by D. Heynderickx and M. Kruglanski
    September 16, 1997 - Bug fixed in version 1.07
