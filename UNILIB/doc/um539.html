UM539

[ Name | Synopsis | Arguments | Description | Diagnostics | Common Blocks | Dependencies | See Also | Reported Bugs | Examples ]

Name

    Subroutine UM539 - Evaluate the magnetic field

Synopsis 

      TYPE(zgeo) mpos
      TYPE(zvec) mb
      REAL*8        amjd
      INTEGER*4     ifail

      CALL UM539(mpos, amjd, mb, ifail)

Arguments

    mpos
    Geographic position in geocentric coordinates (GEO) [in]
    amjd
    Modified Julian day [in]
    mb
    Magnetic field vector [out]
    ifail
    Error flag (passed by dependents) [out]

Description

    The subroutine UM539 evaluates the magnetic field vector at a given time and geographic location. The subroutine differs from subroutine UM530 by its argument amjd from which the time is specified. Note that the magnetic field models have to be firstly initialized, e.g. by a call to the subroutine UM510 and UM520.

    The time (argument amjd) passed to the subroutine is used to re-evaluate the position of the Sun and the GSM or SM coordinate transformation. Note that only the external magnetic field model is affected by the re-evaluation.

Diagnostics

    None

Common Blocks

Dependencies

Called by

    None

Calls

    UM523, GEO to GSM transformation
    UM524, GEO to SM transformation
    UM530, evaluate the magnetic field vector
